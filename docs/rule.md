## rule 1: Xuất log
* Hãy luôn cập nhật log một cách chi tiết vào `THIGIACMAYTINH\log` những gì thêm, xóa, dòng code được sửa code và docs trong dự án
## rule 2: cấu trúc file plan
* Các file plan được lưu ở `THIGIACMAYTINH\docs\plan`
* File plan nên có đủ các mục sau:
    * Goal: mục đích của chức năng được thực hiện bới plan.
    * Input: mô tả input của chức năng.
    * Output: mô tả output của chức năng.
    * How to do: mô tả chức năng được thực hiện như thế nào.
    * Lưu ý (nếu có).
* File plan phải đáp ứng được:
    * Mô tả đủ chi tiết, cung cấp đủ ngữ cảnh để thực hiện.
    * Các file plan không quá 200 dòng.
